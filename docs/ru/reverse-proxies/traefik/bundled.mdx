В этом руководстве мы рассмотрим настройку обратного прокси Traefik.

<Warning>
    Установка на одном сервере означает, что вы установите Traefik на тот же сервер, что и панель Remnawave.
</Warning>

<Danger>
    У вас уже должен быть установлен и настроен обратный прокси Traefik на сервере с панелью Remnawave и остальными компонентами.
</Danger>

## Подготовка нового файла динамической конфигурации

Если Traefik уже настроен, вам нужно создать новый файл динамической конфигурации <Badge>remnashop.yml</Badge> в директории <Badge>/opt/remnawave/traefik/config</Badge>:

<CodeGroup>
    ```bash Конфигурация Traefik
    cd /opt/remnawave/traefik/config && nano remnashop.yml
    ```
</CodeGroup>

Вставьте следующую конфигурацию.

<Warning>
    Замените REPLACE_WITH_YOUR_DOMAIN_BOT на ваше доменное имя.
    Ознакомьтесь с конфигурацией ниже и обратите внимание на выделенные строки.
</Warning>

<CodeGroup>
    ```javascript remnashop.yml highlight={4}
    http:
    routers:
        remnashop:
        rule: "Host(`REPLACE_WITH_YOUR_DOMAIN_BOT`) && PathPrefix(`/api/v1`)"
        entrypoints:
            - https
        tls:
            certResolver: letsencrypt
        service: remnashop

    services:
        remnashop:
        loadBalancer:
            servers:
            - url: "http://remnashop:5000"
    ```
</CodeGroup>

Теперь перезапустите контейнеры Remnashop:

<CodeGroup>
    ```bash Перезапуск контейнеров Remnashop
    cd /opt/remnashop && docker compose down && docker compose up -d && docker compose logs -f
    ```
</CodeGroup>

### Если всё настроено правильно, Remnashop отправит уведомление об успешном запуске в Telegram.
