В этом руководстве мы рассмотрим настройку обратного прокси Nginx.

<Warning>
  **Внимание**<br />
  Установка на одном сервере означает, что вы установите Nginx на тот же сервер, что и панель Remnawave.
</Warning>

<Danger>
  **Важно!**<br />
  У вас уже должен быть установлен и настроен обратный прокси Nginx на сервере с панелью Remnawave и остальными компонентами.
</Danger>

## Подготовка конфигурационного файла Nginx

Если Nginx уже настроен, вам нужно лишь добавить новый блок location в ваш конфигурационный файл.

Выпустите сертификат для доменного имени Remnashop:

<CodeGroup>
```bash Выпуск сертификата
acme.sh --issue --standalone -d 'BOT_DOMAIN' --key-file /opt/remnawave/nginx/botdomain_privkey.key --fullchain-file /opt/remnawave/nginx/botdomain_fullchain.pem --alpn --tlsport 8443
```
</CodeGroup>

Откройте конфигурационный файл Nginx:

<CodeGroup>
```bash Редактирование nginx.conf
cd /opt/remnawave/nginx && nano nginx.conf
```
</CodeGroup>

<Warning>
  **Внимание**<br />
  Замените BOT_DOMAIN на ваше доменное имя.<br />
  Ознакомьтесь с конфигурацией ниже и обратите внимание на выделенные строки.
</Warning>

<Danger>
  **Важно!**<br />
  Не заменяйте существующую конфигурацию полностью — только добавьте новый блок в существующий файл Nginx.
</Danger>

Добавьте новый блок upstream в начало конфигурационного файла. Обратите внимание на выделенные строки — именно их нужно добавить.

<CodeGroup>
```javascript nginx.conf highlight={9-11}
upstream remnawave {
    server remnawave:3000;
}

upstream remnawave-subscription-page {
    server remnawave-subscription-page:3010;
}

upstream remnashop {
    server remnashop:5000;
}
```
</CodeGroup>

Теперь добавьте новый блок server в конец конфигурационного файла:

<CodeGroup>
```bash nginx.conf highlight={2}
server {
    server_name BOT_DOMAIN;

    listen 443 ssl;
    listen [::]:443 ssl;
    http2 on;

    location /api/v1 {
        proxy_http_version 1.1;
        proxy_pass http://remnashop;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header X-Forwarded-Host $host;
        proxy_set_header X-Forwarded-Port $server_port;

        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }

    # Конфигурация SSL (рекомендации Mozilla Intermediate)
    ssl_protocols          TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384:ECDHE-ECDSA-CHACHA20-POLY1305:ECDHE-RSA-CHACHA20-POLY1305:DHE-RSA-AES128-GCM-SHA256:DHE-RSA-AES256-GCM-SHA384:DHE-RSA-CHACHA20-POLY1305;

    ssl_session_timeout 1d;
    ssl_session_cache shared:MozSSL:10m;
    ssl_session_tickets    off;
    ssl_certificate "/etc/nginx/ssl/botdomain_fullchain.pem";
    ssl_certificate_key "/etc/nginx/ssl/botdomain_privkey.key";
    ssl_trusted_certificate "/etc/nginx/ssl/botdomain_fullchain.pem";

    ssl_stapling           on;
    ssl_stapling_verify    on;
    resolver               1.1.1.1 1.0.0.1 8.8.8.8 8.8.4.4 208.67.222.222 208.67.220.220 valid=60s;
    resolver_timeout       2s;

    # Сжатие Gzip
    gzip on;
    gzip_vary on;
    gzip_proxied any;
    gzip_comp_level 6;
    gzip_buffers 16 8k;
    gzip_http_version 1.1;
    gzip_min_length 256;
    gzip_types
        application/atom+xml
        application/geo+json
        application/javascript
        application/x-javascript
        application/json
        application/ld+json
        application/manifest+json
        application/rdf+xml
        application/rss+xml
        application/xhtml+xml
        application/xml
        font/eot
        font/otf
        font/ttf
        image/svg+xml
        text/css
        text/javascript
        text/plain
        text/xml;
}
```
</CodeGroup>

Теперь обновите файл docker-compose.yml для Nginx, чтобы подключить новые файлы сертификата:

<CodeGroup>
```bash Редактирование docker-compose.yml
cd /opt/remnawave/nginx && nano docker-compose.yml
```
</CodeGroup>

<CodeGroup>
```javascript docker-compose.yml highlight={10-11}
services:
    remnawave-nginx:
        image: nginx:1.28
        container_name: remnawave-nginx
        hostname: remnawave-nginx
        volumes:
            - ./nginx.conf:/etc/nginx/conf.d/default.conf:ro
            - ./fullchain.pem:/etc/nginx/ssl/fullchain.pem:ro
            - ./privkey.key:/etc/nginx/ssl/privkey.key:ro
            - ./botdomain_fullchain.pem:/etc/nginx/ssl/botdomain_fullchain.pem:ro
            - ./botdomain_privkey.key:/etc/nginx/ssl/botdomain_privkey.key:ro
        restart: always
        ports:
            - '0.0.0.0:443:443'
        networks:
            - remnawave-network

networks:
    remnawave-network:
        name: remnawave-network
        driver: bridge
        external: true
```
</CodeGroup>

Теперь перезапустите контейнер Nginx:

<CodeGroup>
```bash Перезапуск контейнера Nginx
docker compose down && docker compose up -d && docker compose logs -f
```
</CodeGroup>

И перезапустите контейнеры Remnashop:

<CodeGroup>
```bash Перезапуск контейнеров Remnashop
cd /opt/remnashop && docker compose down && docker compose up -d && docker compose logs -f
```
</CodeGroup>

### Если всё настроено правильно, Remnashop отправит уведомление об успешном запуске в Telegram.
