<Warning>
    Installing on a separate server means that Nginx will be installed on a
    different server than the Remnawave panel and other components.
</Warning>

<Danger>
    There should be no reverse proxies installed on this server.
</Danger>

## Issue an SSL certificate

You need to issue an SSL certificate. The easiest way is to use [acme.sh](https://acme.sh/).

## Install dependencies

<CodeGroup>
    ```bash Install dependencies sudo apt-get install cron socat
    ```
</CodeGroup>

<Info>
    You can use any email address to issue the certificate, but it is
    recommended to use a valid one.
</Info>

Replace <Badge>EMAIL</Badge> with your email address:

<CodeGroup>
    ```bash Install acme.sh curl https://get.acme.sh | sh -s email=EMAIL &&
    source ~/.bashrc
    ```
</CodeGroup>

## Create a folder for certificates

<CodeGroup>
    ```bash Create directory mkdir -p /opt/remnawave/nginx && cd
    /opt/remnawave/nginx
    ```
</CodeGroup>

## Issue a certificate

Replace <Badge>BOT_DOMAIN</Badge> with the domain name you want to issue a certificate for.

<Warning>
    Do not use domain zones: .ru, .su, .рф. ZeroSSL does not currently support
    these zones.
</Warning>

<CodeGroup>
    ```bash Issue certificate acme.sh --issue --standalone -d 'BOT_DOMAIN'
    --key-file /opt/remnawave/nginx/privkey.key --fullchain-file
    /opt/remnawave/nginx/fullchain.pem --alpn --tlsport 8443
    ```
</CodeGroup>

<Info>
    Make sure that port <Badge>8443</Badge> is open on your server. It is
    required for certificate issuance.
</Info>

<Frame>
    <img src="/images/issue-cert.webp" />
</Frame>

This confirms that the certificate has been issued. <Badge>Acme.sh</Badge> will automatically renew the certificate every 60 days — just make sure that port 8443 is open and not in use on your server.

## Nginx configuration

Create a <Badge>nginx.conf</Badge> file in the <Badge>/opt/remnawave/nginx</Badge> directory:

<CodeGroup>
    ```bash Create nginx.conf cd /opt/remnawave/nginx && nano nginx.conf
    ```
</CodeGroup>

Paste the following configuration.

<Warning>
    Replace <Badge color="orange">REPLACE_WITH_YOUR_DOMAIN</Badge> with your
    domain name.
    
    Review the configuration below and pay attention to the highlighted lines.
</Warning>

<CodeGroup>
    ```bash nginx.conf highlight={6}
    upstream remnashop {
        server remnashop:5000;
    }

    server {
    server_name REPLACE_WITH_YOUR_DOMAIN;

        listen 443 ssl;
        listen [::]:443 ssl reuseport;
        http2 on;

        location /api/v1 {
            proxy_http_version 1.1;
            proxy_pass http://remnashop;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
            proxy_set_header X-Forwarded-Host $host;
            proxy_set_header X-Forwarded-Port $server_port;

            proxy_send_timeout 60s;
            proxy_read_timeout 60s;
        }

        # SSL Configuration (Mozilla Intermediate Guidelines)
        ssl_protocols          TLSv1.2 TLSv1.3;
        ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384:ECDHE-ECDSA-CHACHA20-POLY1305:ECDHE-RSA-CHACHA20-POLY1305:DHE-RSA-AES128-GCM-SHA256:DHE-RSA-AES256-GCM-SHA384:DHE-RSA-CHACHA20-POLY1305;

        ssl_session_timeout 1d;
        ssl_session_cache shared:MozSSL:10m;
        ssl_session_tickets    off;
        ssl_certificate "/etc/nginx/ssl/fullchain.pem";
        ssl_certificate_key "/etc/nginx/ssl/privkey.key";
        ssl_trusted_certificate "/etc/nginx/ssl/fullchain.pem";

        ssl_stapling           on;
        ssl_stapling_verify    on;
        resolver               1.1.1.1 1.0.0.1 8.8.8.8 8.8.4.4 208.67.222.222 208.67.220.220 valid=60s;
        resolver_timeout       2s;

        # Gzip Compression
        gzip on;
        gzip_vary on;
        gzip_proxied any;
        gzip_comp_level 6;
        gzip_buffers 16 8k;
        gzip_http_version 1.1;
        gzip_min_length 256;
        gzip_types
            application/atom+xml
            application/geo+json
            application/javascript
            application/x-javascript
            application/json
            application/ld+json
            application/manifest+json
            application/rdf+xml
            application/rss+xml
            application/xhtml+xml
            application/xml
            font/eot
            font/otf
            font/ttf
            image/svg+xml
            text/css
            text/javascript
            text/plain
            text/xml;

    }

    server {
    listen 443 ssl default*server;
    listen [::]:443 ssl default_server;
    server_name *;

        ssl_reject_handshake on;

    }

    ```
</CodeGroup>

## Create docker-compose.yml

Create a <Badge>docker-compose.yml</Badge> file in the <Badge>/opt/remnawave/nginx</Badge> directory:

<CodeGroup>
    ```bash Create docker-compose.yml
    cd /opt/remnawave/nginx && nano docker-compose.yml
    ````

    </CodeGroup>

    Paste the following configuration:

    <CodeGroup>
    ```bash docker-compose.yml
    services:
        remnawave-nginx:
            image: nginx:1.28
            container_name: remnawave-nginx
            hostname: remnawave-nginx
            volumes:
                - ./nginx.conf:/etc/nginx/conf.d/default.conf:ro
                - ./fullchain.pem:/etc/nginx/ssl/fullchain.pem:ro
                - ./privkey.key:/etc/nginx/ssl/privkey.key:ro
            restart: always
            ports:
                - '0.0.0.0:443:443'
            networks:
                - remnawave-network

    networks:
    remnawave-network:
    name: remnawave-network
    driver: bridge
    external: true

    ```
</CodeGroup>

## Start the container

<CodeGroup>
    ```bash Start the container
    docker compose up -d && docker compose logs -f -t
    ```
</CodeGroup>

And restart the Remnashop containers:

<CodeGroup>
    ```bash Restart Remnashop containers cd /opt/remnashop && docker compose
    down && docker compose up -d && docker compose logs -f
    ```
</CodeGroup>

### If everything is configured correctly, Remnashop will send a successful launch notification in Telegram.
