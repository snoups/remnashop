title: "Separate server"
description: "Simple configuration"

<Warning>
    **Warning**
    <br />
    Installing on a separate server means that Caddy will be installed on a
    different server than the Remnawave panel and other components.
</Warning>

<Danger>
    **Important!**
    <br />
    There should be no reverse proxies installed on this server.
</Danger>

Create a Caddyfile in the /opt/remnawave/caddy directory:

<CodeGroup>
    ```bash Create Caddyfile mkdir -p /opt/remnawave/caddy && cd
    /opt/remnawave/caddy && nano Caddyfile ```
</CodeGroup>

Paste the following configuration.

<Warning>
    **Warning**
    <br />
    Replace REPLACE_WITH_YOUR_DOMAIN with your domain name.
    <br />
    Review the configuration below and pay attention to the highlighted lines.
</Warning>

<CodeGroup>
```bash Caddyfile highlight={1}
https://REPLACE_WITH_YOUR_DOMAIN {
        reverse_proxy /api/v1/* http://remnashop:5000
}
:443 {
    tls internal
    respond 204
}
```
</CodeGroup>

## Create docker-compose.yml

Create a <Badge>docker-compose.yml</Badge> file in the <Badge>/opt/remnawave/caddy</Badge> directory:

<CodeGroup>
    ```bash Create docker-compose.yml cd /opt/remnawave/caddy && nano
    docker-compose.yml ```
</CodeGroup>

Paste the following configuration:

<CodeGroup>
    ```bash docker-compose.yml services: caddy: image: caddy:2.9 container_name:
    'caddy' hostname: caddy restart: always ports: - '0.0.0.0:443:443' -
    '0.0.0.0:80:80' networks: - remnawave-network volumes: -
    ./Caddyfile:/etc/caddy/Caddyfile - caddy-ssl-data:/data networks:
    remnawave-network: name: remnawave-network driver: bridge external: true
    volumes: caddy-ssl-data: driver: local external: false name: caddy-ssl-data
    ```
</CodeGroup>

## Start the container

<CodeGroup>
    ```bash Start the container docker compose up -d && docker compose logs -f
    -t ```
</CodeGroup>

Now restart the Remnashop containers:

<CodeGroup>
    ```bash Restart Remnashop containers cd /opt/remnashop && docker compose
    down && docker compose up -d && docker compose logs -f ```
</CodeGroup>

### If everything is configured correctly, Remnashop will send a successful launch notification in Telegram.
