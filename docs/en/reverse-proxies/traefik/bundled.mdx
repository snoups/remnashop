In this guide, we will look at the configuration and setup of the Traefik reverse proxy.

<Warning>
    **Warning**
    <br />
    Bundled installation means that you will install Traefik on the same server
    as the Remnawave panel.
</Warning>

<Danger>
    **Important!**
    <br />
    You must already have the Traefik reverse proxy installed and configured on
    your Remnawave panel server and other components.
</Danger>

## Prepare a new dynamic configuration file

If Traefik is already configured, you need to create a new dynamic configuration file <Badge>remnashop.yml</Badge> in the <Badge>/opt/remnawave/traefik/config</Badge> directory:

<CodeGroup>
    ```bash Traefik configuration cd /opt/remnawave/traefik/config && nano
    remnashop.yml ```
</CodeGroup>

Paste the following configuration.

<Warning>
    **Warning**
    <br />
    Replace REPLACE_WITH_YOUR_DOMAIN_BOT with your domain name.
    <br />
    Review the configuration below and pay attention to the highlighted lines.
</Warning>

<CodeGroup>
    ```javascript remnashop.yml highlight={4}
    http: routers: remnashop: rule: "Host(`REPLACE_WITH_YOUR_DOMAIN_BOT`) &&
    PathPrefix(`/api/v1`)" entrypoints: - https tls: certResolver: letsencrypt
    service: remnashop services: remnashop: loadBalancer: servers: - url:
    "http://remnashop:5000" ```
</CodeGroup>

Now restart the Remnashop containers:

<CodeGroup>
    ```bash Restart Remnashop containers cd /opt/remnashop && docker compose
    down && docker compose up -d && docker compose logs -f ```
</CodeGroup>

### If everything is configured correctly, Remnashop will send a successful launch notification in Telegram.
